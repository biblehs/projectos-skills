Use this checklist when creating or refreshing `ops/RELEASE_READINESS.md`.

## Required Sections

- intended release slice
- ready now
- blockers
- risky but optional items
- deferred work
- verification plan
- rollback or containment plan
- go or no-go call

## Review Questions

- What exactly is intended to ship in this release?
- Which user journeys must pass for the release to count as viable?
- Which items are hard blockers versus cleanup or polish?
- Which known risks can be tolerated for this release?
- How will the release be verified after publishing?
- What is the rollback or containment path if problems appear?
