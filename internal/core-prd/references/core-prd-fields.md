# Core PRD Fields

Use these sections as the default shape for `PRD_CORE.md`.

## Recommended Sections

1. One-sentence product definition
2. Why now
3. Primary early user
4. Core problem or job to be done
5. Primary user value
6. Product promise and non-goals
7. First validating user flow
8. Core object or system shape for MVP
9. Product principles
10. In scope now
11. Explicitly out of scope now
12. Assumptions, constraints, and dependencies
13. Success criteria
14. Open questions
15. Canonical questions before new work

## Quality Bar

A good core PRD:

- is short enough to read in one sitting
- states the product in human language
- defines what the product is not
- explains why the current release matters now
- contains a concrete first user flow, not only feature bullets
- narrows current version scope
- makes assumptions and open questions visible
- helps design, architecture, and execution stay aligned

## Common Failure Modes

- feature lists without product meaning
- broad long-term ambition with no current version boundary
- multiple conflicting product framings
- no statement of what is out of scope
- no first validating flow
- no assumptions or unresolved questions, causing downstream layers to invent them
