# Project Master Fields

Use these sections as the default shape for `PROJECT_MASTER.md`.

## Recommended Core Sections

1. Project definition
2. What the project is not
3. Current stage
4. Current version focus
5. Milestones
6. Module status overview
7. This week's recommended focus
8. Current risks
9. Recent critical decisions
10. Key links
11. Management rule

## Quality Bar

A good project master:

- is readable in under five minutes
- tells the builder what matters now
- reduces ambiguity instead of adding more
- points clearly to deeper docs
- stays updated as the project changes
- still shows one center of gravity

## Common Failure Modes

- too much detail
- vague milestone names
- duplicated status from multiple sources
- mixing long task lists into the master view
- no direct links to the current source documents
