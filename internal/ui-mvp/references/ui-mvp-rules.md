Use this guide when creating or refreshing `design/UI_MVP.md`.

## Core Principle

The UI MVP should prove the product promise with the smallest credible interface, not merely the smallest amount of work.

## Required Sections

- MVP objective
- target user moment
- primary validating flow
- primary surface chosen from product objects and user jobs
- minimum screen list
- essential states and trust points
- visual thesis and first-screen direction
- reference-site takeaways when the user provides websites
- what is intentionally excluded
- open design questions

## Review Questions

- Which user journey most directly validates the product?
- What is the product's core object: transaction, document, task, message, account, route, event, record, asset, or relationship?
- What surface best fits that object and user job: ledger, planner, calendar, timeline, editor, map, chat, workflow board, review queue, canvas, or hybrid?
- Which screens are required to complete that journey?
- Which states must exist for the flow to feel trustworthy?
- Which surfaces can be deferred without breaking validation?
- Where does the interface need clarity more than polish?
- If a reference website is provided, which visual principles should be adapted and which brand-specific details must be replaced?
