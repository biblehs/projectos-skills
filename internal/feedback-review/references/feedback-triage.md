# Feedback Triage

Use this guide to turn raw feedback into local product decisions.

## Signal Categories

### Core-product signal

Changes what the product should fundamentally do or how users experience the central value.

### UX friction signal

The product is directionally right, but users are getting stuck, confused, or unconvinced.

### Support-layer signal

Access, billing, onboarding friction, routing issues, trust issues, or communication gaps around the core product.

### Noise or low-confidence signal

Single anecdotes, requests outside the product center, or signals without enough evidence.

## Questions To Ask

1. Is this repeated or isolated?
2. Does this change the product core or only the presentation?
3. Should this affect the current sprint, later planning notes, or only the backlog?
4. Does this suggest a PRD change, a UI change, or a release-readiness change?
5. If several local work tracks are active, which lane should absorb the change?

## Output Recommendation

For each meaningful signal, record:

- source
- category
- confidence
- recommended action
- related docs or modules
- related work track when relevant
