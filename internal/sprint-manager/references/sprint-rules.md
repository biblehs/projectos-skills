# Sprint Rules

## Default Sections

1. Current sprint objective
2. In progress
3. Next up
4. Blockers
5. Backlog
6. Sprint rules
7. Weekly check-in questions

## Quality Bar

A good sprint board:

- has one clear objective
- limits concurrent work
- separates current work from later work
- makes blockers visible
- maps tasks to modules or milestones when possible
- keeps parallel work subordinate to one core sprint objective

## Anti-Patterns

- too many in-progress items
- long-term planning items copied into sprint without filtering
- support work consuming the sprint without being named as such
- backlog used as a second in-progress list
- too many parallel tracks treated as top priority
